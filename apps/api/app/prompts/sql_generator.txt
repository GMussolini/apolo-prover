INSTRUÇÕES:
1. Para cada ramo, gere o WHERE substituindo o placeholder. Use "1=1" se a pergunta não filtra esse ramo.
2. `select_list` — colunas do resultado (pode incluir agregações como COUNT(*), SUM(col_agregavel), AVG, MIN, MAX nas colunas marcadas [agregavel]).
3. `group_by` — colunas marcadas [dimensao] quando houver agregação. Vazio se não tem agregação.
4. `order_by` — coluna + ASC/DESC. Vazio se irrelevante.
5. `top` — limite de linhas. Default 500. Para "quantos..." é 1 (COUNT retorna 1 linha). Para "top 10..." é 10.
6. Datas em formato ISO 'YYYY-MM-DD'. Use GETDATE() pra "hoje".
7. Strings em LIKE '%valor%' (case-insensitive por padrão no SQL Server collation pt-BR).
8. NUNCA escreva FROM/JOIN/UNION/SUBQUERY. NUNCA referencie tabelas pelo nome — só os aliases das colunas declaradas acima.
